# Everything Claude Code (ECC) — Patterns Worth Borrowing

Reference notes extracted before deleting `Donors/new/everything-claude-code/` (April 2026).
Upstream: `github.com/chernistry/everything-claude-code` (plugin repo with ~70 agents, ~30 slash commands, multi-IDE scaffolding, gated hooks).

Most of ECC overlaps with Forgemaster's existing agent/skill library and is not worth re-absorbing. The patterns below are the ones Forgemaster genuinely lacks.

---

## 1. Runtime hook gating via env vars + wrapper script
**Upstream paths:** `scripts/hooks/run-with-flags.js`, `hooks/hooks.json`

Every hook is invoked through a wrapper that consults `ECC_HOOK_PROFILE` (`minimal|standard|strict`) and `ECC_DISABLED_HOOKS` before running. Users toggle hooks without editing `settings.json`.

Example entry:
```json
{ "command": "node \"${CLAUDE_PLUGIN_ROOT}/scripts/hooks/run-with-flags.js\" \"pre:bash:block-no-verify\" \"scripts/hooks/block-no-verify.js\" \"minimal,standard,strict\"" }
```

Relevant when NOVA adds its planned SessionStart/PreCompact hooks (see `docs/ROADMAP.md` "Claude Code hooks integration") — ship them behind the same gating pattern so users can disable per-profile.

## 2. Hook authoring discipline
**Upstream paths:** `.claude/rules/node.md`, inline comments across `scripts/hooks/*.js`

Stated rules worth adopting verbatim for any NOVA hook:
- Blocking hooks (PreToolUse, Stop) must finish in <200ms — no network calls.
- Always `exit 0` on parse errors; log to stderr with a bracketed prefix (`[HookName] ...`).
- Keep each hook script <200 lines; extract helpers to a shared lib dir.
- Async hooks declare `"async": true` in settings with a timeout ≤30s.

## 3. Specific hook ideas worth stealing
**Upstream paths:** `scripts/hooks/*.js`

Forgemaster has no equivalents for these:
- `block-no-verify.js` — refuses `git commit --no-verify` / `--no-gpg-sign`. Already matches NOVA's CLAUDE.md "never skip hooks" rule but enforces it.
- `pre-bash-commit-quality.js` — lint staged files + commit-msg format + secret/`console.log`/`debugger` scan before a commit runs.
- `mcp-health-check.js` — pings configured MCP servers, reports which are down.
- `cost-tracker.js` — token/cost accounting per session (complements NOVA's `nova_usage.jsonl` and the cost-tracking item in `BERNSTEIN_PATTERNS.md`).
- `pre-compact.js` — parallel track to the PreCompact block-until-saved pattern already rolled into the ROADMAP hooks bullet.
- `session-activity-tracker.js` — passive telemetry of tool usage within a session.

## 4. Language-scoped rules folders
**Upstream path:** `rules/{cpp,csharp,dart,golang,java,kotlin,perl,php,python,rust,swift,typescript,web,common}/`

Per-language rule bundles sitting alongside a `common/` bundle. Forgemaster's `agents/` is divided by role/division, not by language — a thin `rules/{lang}/` tree would give language-specific style/testing/security conventions a clean home that's separate from agent personas. Cheap to add.

## 5. Multi-CLI scaffolding in one repo
**Upstream paths:** `.claude/`, `.codex/`, `.cursor/`, `.gemini/`, `.kiro/`, `.opencode/`, `.trae/`

Same content surfaced through config dirs for seven different agent CLIs. If Forgemaster ever ships as a plugin/profile for non-Claude agents (pairs with the Bernstein adapter pattern, `BERNSTEIN_PATTERNS.md` item 2), this is the directory-layout reference.

## 6. Meta-document separation
**Upstream files:** `AGENTS.md`, `CLAUDE.md`, `RULES.md`, `SOUL.md`, `WORKING-CONTEXT.md`, `EVALUATION.md`, `REPO-ASSESSMENT.md`, `COMMANDS-QUICK-REF.md`, `TROUBLESHOOTING.md`

Concerns split into distinct top-level docs instead of one monolithic README. Worth eyeballing before the next round of NOVA doc cleanup — some (EVALUATION, TROUBLESHOOTING, COMMANDS-QUICK-REF) are patterns NOVA could use.

## 7. Auto-generated guardrails
**Upstream path:** `.claude/rules/everything-claude-code-guardrails.md`

Rules file generated by walking repo history (commit style, module layout, test layout, naming, recommended install profile, detected workflows). The file carries an explicit "regenerate when conventions materially change" note. Relevant to NOVA's `nova_evolve` tool — a dedicated "regenerate guardrails from repo history" command would be a concrete target for it.

## 8. `/learn` and `/skill-create` commands
**Upstream paths:** `commands/` (various)

`/learn` extracts patterns from the current session into reusable form; `/skill-create` generates a skill scaffold from git history. Forgemaster has no skill-bootstrap command — everything authored by hand. Worth a small command that diffs recent commits and drafts a skill stub in `forgemaster/library/`.

---

## Deliberately not borrowed

- The ~70 ECC agents — Forgemaster's `agents/` (326 personas, 18 divisions) is larger and better organized.
- The ~30 slash commands — most overlap with existing Forgemaster skills.
- `ecc_dashboard.py` — 40KB operational dashboard; NOVA's dashboard story belongs with the planned VS Code extension (see ROADMAP), not a standalone Python TUI.
